Auto Fill SABR Worksheet Version II

Version date: 2026-05-11

PlanEvaluation reads an exported DVH file, matches plan elements to a selected SABR report definition, and writes values into an Excel worksheet template for review and documentation.

1) Save the DVH Data

  1. Open the DVH view in External Beam Planning.
  2. In the "Dose Statistics" Tab Right-Click and select "Show DVH for all Structures".
  3. DVH values will be calculated.
  4. Right-Click within the DVH and select "Absolute Dose".
  5. Right-Click within the DVH and select "Export DVH in Tabular Format...".
  6. A "Save As" dialogue box will appear.
  7. Select the folder "M:\Dosimetry Planning Documents\SABR Plan Evaluation\DVH Files".
  8. Set "Save as type" to "All Files".
  9. Save with a file name ending in ".dvh".
Confirm the selected DVH export is the current approved plan and fractionation before running the report.

2) Run the Plan Evaluation program

Program name

  • PlanEvaluation

Start method

  • Run PlanEvaluation.bat in the package root.
  • Run PlanEvaluation.py from the same folder with the configured Python environment.

On startup, the app loads configuration from PlanEvaluationConfig.xml, report definitions from Data/Reports.pkl, and available DVH plans from the configured DVH folder.

3) Main Window

After starting "PlanEvaluation", the main window appears.

  1. Select the dvh file from the list in the bottom left of the window. (This list reflects dvh files in the configured DVH directory.)
  2. To select a DVH file from a different folder, use File > Select Plan DVH File. A dialog will appear — type or browse to the file path and click Ok.
  3. Click Load Plan to import the dvh file data.
  4. The button label should change to Plan Loaded.
  5. Select a report from the report drop-down list.
  6. Click Match Structures to run automatic matching and review manual matching.

4) Matching Structures And Values

In the manual matching window, unmatched items can be manually mapped, directly entered as values, or cleared.

After clicking Match Structures, a new window will pop up allowing you to match the plan report items with structure names.

The items are grouped by whether the auto-match was successful or not.

Matching tries these steps in order for each report reference:

  1. Direct name match to plan element.
  2. Laterality-based name variation (using config lookup tables).
  3. Alias lookup from global alias list and per-report aliases.

Manual match actions

  • To manually match a structure to a given report item: right click on the desired item and hold the mouse over the Match heading on the menu that appears.
  • Click on the appropriate structure from the list.
  • Selecting Enter Value from the right-click menu brings up a new window for typing in the text value.
  • The value entered will display in the Matched Item column.
  • To remove a match, select Clear from the right-click menu.
  • The value in the Matched Item column will become None.
  • Once all desired modifications are complete, click on the Approve button in the lower left corner.

5) Generate And Review Report

  1. Click Generate Report.
  2. The SABR Plan Report Form will appear and begin to populate.
  3. Review Results.

    4. Note: values for both Trachea and Bronchus may appear, with the maximum used in the usual spot.

  4. Clicking on any of the green buttons, or selecting a new dvh file or plan report, moves back to that step in the process.

6) Menus You Might Want to Use

Menu Action Purpose
Select Plan DVH File Open a specific DVH file directly when it does not appear in the list of available files.
Set Save File Name Choose a folder and filename for the generated report.

7) Menu Items You Should Not Use

Menu Action Purpose
Set Default Locations Change DVH folder, report folders, template and save locations in config. This will change the locations for all users and may cause issues if set incorrectly. Contact a physicist if you need changes to these.
Update all Report Definitions Rebuild report definition pickle from configured report folders. This is not needed for normal use and may cause issues if the report definitions are not correct.
Load Report Definition File Load an additional report definition XML file. This is primarily intended for physicists to test new report definition XML files. If a new report definition XML file is needed, contact a physicist to create and test it.

8) Troubleshooting

Invalid Plan

  • Cause: selected file is not valid DVH format or cannot be parsed.
  • Action: verify extension and source export, then re-select file.

Missing Match Items

  • Cause: naming mismatch between DVH structures and report references.
  • Action: use manual matching, then update aliases for future automation.

Excel File Does Not Update

  • Cause: output file is locked or workbook path is incorrect.
  • Action: close conflicting workbook instances and verify save path.
